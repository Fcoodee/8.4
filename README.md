## ملاحظات مهمة قبل الرفع على GitHub

1. **`dev-access.txt` يحتوي بيانات دخول بنص صريح (admin:2030)**. إذا رُفع المستودع كـ **عام
   (Public)** فسيتمكن أي شخص من قراءته مباشرة عبر الرابط. يُنصح بشدة إما إبقاء المستودع
   **خاصًا (Private)**، أو نقل هذا الملف لموضع لا يُقرأ من المتصفح مباشرة، أو قبول أنها حماية
   شكلية فقط (كما هو موضّح داخل صندوق حوار هوية المطوّر نفسه).
2. **جلب `dev-access.txt` عبر `fetch`** يعمل فقط عند تشغيل الموقع عبر خادم (http/https) — لن
   يعمل عند فتح الملفات مباشرة من القرص (`file://`)، ويعمل بشكل طبيعي على GitHub Pages.
3. **كود الجافاسكربت الداخلي في `index.html` و`report-v7.html` مُعمًّى تعمية خفيفة** (Base64 +
   فك تشفير وقت التشغيل) — لا يمنع مبرمجًا متمرسًا من قراءته، فقط يُبعده عن "عرض المصدر" المباشر.
   ملفات `faq-widget.js` و`update-banner.js` **غير مُعمّاة** عمدًا لسهولة الصيانة.
4. **زر "راسل المطوّر"** يعتمد على بروتوكول `mailto:` — يفتح برنامج البريد المُثبَّت على جهاز
   الزائر (Outlook، Gmail عبر المتصفح، تطبيق البريد بالجوال، إلخ) حسب إعدادات نظامه.
5. **`corp-comm-proposal.html`** ملف كبير الحجم نسبيًا (يحتوي صورًا مُرمَّزة بصيغة Base64) — هذا
   طبيعي ومطابق لما أُرسل أصلًا، ويحتاج مرجعًا خارجيًا لـ `logo.png` غير مُضمَّن في الحزمة (سيظهر
   كصورة مكسورة إن لم يُرفق الشعار بنفس الاسم في نفس المجلد).
6. جميع الروابط الداخلية بين الملفات نسبية ولا تحتاج تعديلًا عند الرفع، طالما بقيت كل الملفات
   التسعة في نفس المجلد الجذري للمستودع.
