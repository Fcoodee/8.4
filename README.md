   الزوّار. إن أردتم تحليلًا حقيقيًا مُجمَّعًا لاحقًا، هذا يتطلّب خادمًا فعليًا (خارج نطاق موقع استاتيكي).

## ⚠️ تنبيه مهم جدًا — افتح الملفات من نفس المجلد دائمًا

الشات بوت وبانر التحديث (وحتى بيانات دخول المطوّر) تعتمد على ملفات خارجية مرتبطة بمسارات نسبية
(`faq-widget.js`، `update-banner.js`، `dev-access.txt`). **إذا فتحتَ ملف HTML واحدًا بمفرده خارج
هذا المجلد، لن تظهر هذه الميزات ولن تُصدر أي رسالة خطأ ظاهرة للمستخدم العادي** — الفشل صامت.
تأكد دائمًا من فكّ ضغط الحزمة كاملة، وفتح `index.html` من داخل نفس المجلد الذي يحوي كل الملفات
الثمانية معًا.

## ملاحظات مهمة قبل الرفع على GitHub

1. **`dev-access.txt` يحتوي بيانات دخول بنص صريح (admin:2030)**. إذا رُفع المستودع كـ **عام (Public)**
   فسيتمكن أي شخص من قراءته مباشرة عبر الرابط. يُنصح بشدة إما:
   - إبقاء المستودع **خاصًا (Private)**، أو
   - نقل هذا الملف لموضع لا يُقرأ من المتصفح مباشرة (يتطلب خادمًا فعليًا لا استضافة استاتيكية كـ GitHub Pages)، أو
   - قبول أنها حماية شكلية فقط (كما هو موضّح داخل نافذة الدخول نفسها في الموقع).

2. **جلب `dev-access.txt` عبر `fetch`** يعمل فقط عند تشغيل الموقع عبر خادم (http/https) — لن يعمل
   عند فتح الملفات مباشرة من القرص (`file://`)، ويعمل بشكل طبيعي على GitHub Pages — يُفضَّل اختبارها
   بعد الرفع مباشرة للتأكد.

3. **كود الجافاسكربت الداخلي في `index.html` و `report-v7.html` مُعمًّى تعمية خفيفة** (Base64 + فك
   تشفير وقت التشغيل) — هذا لا يمنع مبرمجًا متمرسًا من قراءته، فقط يُبعده عن "عرض المصدر" المباشر.
   ملف `faq-widget.js` **غير مُعمًّى** عمدًا لسهولة الصيانة والتحديث.

4. **سجل محاولات الدخول المحلي** (`localStorage`) يُمسَح تلقائيًا إذا مسح الزائر بيانات المتصفح، ولا
   يظهر إلا لمن يملك صلاحية الدخول لصفحة `brainstorm-database.html` على نفس المتصفح الذي جرت منه
   المحاولات.

5. جميع الروابط الداخلية بين الملفات نسبية ولا تحتاج تعديلًا عند الرفع، طالما بقيت كل الملفات
   (بما فيها `faq-widget.js`) في نفس المجلد الجذري للمستودع.

6. **زر "تحميل التقرير بصيغة PDF"** في `index.html` يفتح `report-v7.html?print=1` في نافذة جديدة
   ويُشغّل نافذة طباعة المتصفح تلقائيًا — يعمل في كل المتصفحات الحديثة دون أي إعداد إضافي.
